# Jira Basics (Quick Notes)

## What Jira is used for
- Tracking work items (bugs, stories, tasks)
- Assigning ownership and status
- Prioritizing work and planning sprints
- Maintaining traceability: requirements → tests → bugs → fixes

## Core fields you should always complete for a Bug
- **Summary**: short, specific title
- **Description**: environment + preconditions + steps + expected vs actual
- **Evidence**: screenshot/video
- **Labels**: searchable keywords (3–5 max)
- **Severity / Priority** (if available)

## Good bug title pattern
`<feature> <problem> <condition>`
Example: `Sign in button stuck with valid credentials (Incognito)`

## Where recruiters look
- Clear reproduction steps
- Clean evidence naming
- Consistent documentation
